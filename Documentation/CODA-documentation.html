1. Introduction

About CODA

CODA stands for COvid-19 Data for All. It is a smart micro-service-based web system that is able to provide information regarding the Covid-19 pandemic in multiple written and visual formats for both specialists and the general public. Every data collected by CODA such as articles, documents and graphics, contains attested pieces of information which can be easily viewed, downloaded and shared to other platforms. Moreover, everything on CODA can be accessed via a SPARQL endpoint created in order to ease user access to required knowledge. CODA’s goal is to keep everyone informed about the pandemic in a fast and easy manner and avoid spreading misinformation to the general public.

3. Microservices

Data Crawler

A data crawler or a web crawler is an Internet bot that systematically browses the World Wide Web, typically for creating search engine indices. CODA makes use of its own personalised data crawler that collects Covid-19-related data from trusted news sources all over the internet. The Scrapy framework for Python is used in order to create this specific web crawler. The sources of information that are scraped by the data crawler come from official news articles sites, statistics sites and many more. A few examples are The Guardian, The New York Times and BBC News.

Data Visualiser

CODA wants to offer its users a platform that can speed up the process of analysis and understanding. The Coronavirus pandemic deals with many rapidly-changing variables and putting them into perspective can offer a better image of the evolution of the event. Thus, CODA makes great use of different graphics and statistics that show the numbers involved in the Pandemic. With the help of Google Charts and the COVID19 API developed by Postman, CODA manages is able to display the following data:

  • Number of cases by continent/country/city etc.
  • Number of new cases
  • Number of deaths
  • Active cases
  • Closed cases
Any visual data available on CODA can also be downloaded in CSV or JSON format.

Data Study

This microservice deals with analysing scientific documents from different sites and creating summaries about the articles found on there. The essential data extracted from those articles is used to create suggestion boxes that are displayed in most pages of CODA. When selected, the suggestion will lead to the original page of the article. This is done in order to keep the user interested in reading more information.

Data Sharing

Logged users as well as guest users are able to share any data present on CODA to the available platforms (Facebook, Twitter, Messenger, WhatsApp, Slack etc.) or simply by copying and sending the link.

4. Implementation Logic

CODA’s implementation makes great use of APIs for both the user interaction and the back-end development. The entire website is stored in Cloud via Amazon Web Services.

Front-End

Design-wise, the front end uses the React Framework as well as a few other JavaScript libraries mentioned in the Technology Stack section of the documentation such as Google Charts.

When accessing the main page of CODA, the user is presented with a menu bar, log-in/sign-in buttons, a list of the most popular data collected by CODA over the past 24 hours and a “Recommended” bar with different articles curated by the Data Study microservice. The menu bar offers access to other services available on the site such as the Data visualiser and the public API.

The Data Visualiser page offers a set of pre-defined graphs showcasing the most sought after data related to the pandemic (Current number of cases, Total number of cases etc.), data which is updated în real time.

Log-in and Sign-in buttons are for authentication purposes and prompt the user with an authentication pop-up.

Back-End

The back-end consists of microservices, an API that is accessible by both the backend and the users directly.

Data Crawler is used to gather articles from a list of trusted websites. The articles are then displayed by the front-end in the Articles section of the main web-page.

Data Visualiser gathers updated COVID-19 numbers via an API call to Covid19Api. The collected data is then processed and sent to the front-end where Google Charts is used to visually display the numbers.

Data Study is used to collect articles from trusted medical sources and extract the most important data from them. After this process, the source URL, title and the resulting summary is sent to the front-end where it is added in the “Recommended” section of the page.

Data Sharing is the service which facilitates communication between CODA and selected Social media platforms in order to distribute data more easily. Data Sharing is also capable of creating shortened URLs in case the user does not want to share data directly to other platforms or the desired platform is not available.

CodAPI, the API developed for CODA can be used both by the regular user as well as the back-end. This API offers the options to get various data on COVID19 numbers or articles present on the website as well as download it in CSV and JSON format. If the user wishes to see data on countries they can get the list of countries with available data, latest cases counts for a country, total cases in a day and in a date range. This data can be further filtered into confirmed cases, cured cases or number of tests. For news related to the pandemic, the user can request the latest news, filter news by country, by date range or by keywords. Admins can delete news via the API, în which case token authorization is required. If article data is desired, the user can use a request to get the latest article, filter them by type (dataset, journal, report etc.) and so on. Just as for news articles, admins can delete articles if they possess an authorization token.

5. User Journey

CODA allows both authenticated and unauthenticated usage. There are three types of users that can use the site, each with varying degrees of accessibility.

Guest User

Guest users can browse the website and view every data available on it as well as use the SPARQL endpoint.

Logged User

Logged users have the same accesses as Guest users plus a few added benefits. Registered users can favourite an article, leave a comment, report an article and save queries.

Admin

Admins monitor the site and its activity. Administrators can receive reports and review them, remove, add or edit data on CODA.
